{# ========================= SLA ========================= #}

Objetivos de SLA (minutos)

Se utilizan para evaluar desempeño y disparar alertas por demora (espera, descarga y total).

Recomendación: si dejas vacío un SLA, el sistema lo considera “sin objetivo” (no dispara alerta por ese tramo).

{# ========================= INCIDENCIAS CRÍTICAS ========================= #}

Incidencias críticas

Umbral de incidencias de criticidad alta por recepción.

Si el valor es 0, cualquier incidencia “alta” generará alerta (si está habilitado).

{# ========================= VALIDACIONES DE CAPTURA (enterprise) ========================= #}

Validaciones de captura (líneas inbound)

Define qué campos son obligatorios al crear líneas (lote/vencimiento/temperatura). Esto alinea UI + backend (services) para evitar datos incompletos.

Nota: estas reglas son “enterprise”: ayudan a estandarizar calidad de datos para analítica, auditoría y ML.
